# HH_SSAR
**HeadHunter Semantic Similarity Analysis for Resume**  
*Анализ семантического сходства резюме с вакансиями на HH.ru*

Система семантического поиска релевантных вакансий по тексту резюме с использованием технологий машинного обучения и обработки естественного языка. Проект реализует современные подходы NLP для анализа соответствия резюме требованиям вакансий.

## Цель проекта
Повышение эффективности поиска работы за счёт:
- Автоматического анализа семантической близости резюме и вакансий
- Рекомендаций по оптимизации резюме под конкретные позиции
- Ранжирования вакансий по релевантности для конкретного кандидата
- Выявления ключевых навыков и требований рынка труда


## Основные возможности
### Сбор и анализ данных
- **Автоматический парсинг** вакансий с HeadHunter.ru
- **Извлечение резюме** из открытых источников
- **Обработка больших объемов** данных с кэшированием

### Машинное обучение
- **Word2Vec модели** на базе корпуса Википедии (300+ миллионов токенов)
- **Семантическая векторизация** текстов резюме и вакансий
- **Косинусное сходство** для измерения релевантности

### Анализ и рекомендации
- **Scoring система** оценки соответствия резюме вакансии
- **Топ-N рекомендации** наиболее подходящих позиций

### Основные модули
#### 1. Модуль HH
Сбор данных с HeadHunter.ru:
- Выгрузка вакансий по регионам и ключевым словам
- Парсинг открытых резюме
- Сохранение данных в CSV

#### 2. Модуль Word2Vec
Обучение векторных представлений:
- Загрузка и обработка датасетов Википедии
- Обучение Word2Vec моделей
- Генерация эмбеддингов для текстов

#### 3. Модуль UltimateMatchingModel
Сопоставление резюме с вакансиями:
- Объединяет семантический и многокритериальный анализ
- Использует взвешенную систему оценок (семантика: 35%, критерии: 25%)
- Вычисляет итоговый score и уровень уверенности в результате
- Ранжирование по релевантности
- Генерация рекомендаций

### Ручной запуск (для разработчиков)

Если предпочитаете ручной запуск:

```bash
# Создание виртуального окружения
python -m venv .venv

# Активация (Windows)
.venv\Scripts\activate

# Установка зависимостей  
pip install -r requirements.txt

# Исправление совместимости
pip uninstall scipy gensim -y
pip install scipy==1.11.4 gensim==4.3.2

# Запуск системы
python main.py
```

### Требования к системе

- **Python 3.8+** - установлен и добавлен в PATH
- **Интернет-соединение** - для скачивания данных и моделей
- **~10 ГБ свободного места** - для корпуса Wikipedia и моделей
- **8+ ГБ RAM** - рекомендуется для обучения моделей

### Решение проблем

**Python не найден:**
```bash
# Убедитесь, что Python установлен из python.org
# и флаг "Add Python to PATH" был отмечен при установке
python --version
```

**Ошибки зависимостей:**
```bash
# Обновите pip до последней версии
python -m pip install --upgrade pip

# Переустановите зависимости
pip install -r requirements.txt --force-reinstall
```

**Недостаточно памяти:**
- Закройте лишние программы
- Уменьшите `vector_size` в настройках Word2Vec
- Используйте меньшее количество файлов Wikipedia
